# <h2 style="color:darkblue; font:Times New Roman;">Phase 2: Arabidopsis Homology / BLASTp — Step 01</h2> 
<br>

## **Aim**  
Build a localized *Arabidopsis thaliana* MIKCc reference database and execute a high-throughput BLASTp search against it to map every uncharacterized target species sequence to its closest, evolutionarily predictable homolog.
<br>

## Steps:-

```bash
#!/bin/bash
```

> ### Step 0: Setup Database
Use the previously downloaded TAIR Arabidopsis sequences (which have already been filtered down to the exact 107 functional ABCDE/MIKCc genes like AP1, AG, PI, FLC, SEP).
```bash
cd /Users/bushrakhan/Desktop/NIPGR-data/NIPGR_WORK
mkdir Arabidopsis_BLAST
cd Arabidopsis_BLAST

wget "[Assuming TAIR filtered subset AT_ALL_MADSbox_PROTEINseq.txt.fa is provided]"

makeblastdb -in AT_ALL_MADSbox_PROTEINseq.txt.fa \
            -dbtype prot \
            -parse_seqids \
            -out AT_MADS_ref_db
```

> ### Step 1: Execute `blastp` Against Arabidopsis
*Note: BLASTp against Arabidopsis is the first major inter-species comparative step we are taking in this workflow!*
Query the final deduplicated `MIKCc` sets of our 9 generated species against the anchor database. Below is the example execution specifically for *Amborella trichopoda*.
```bash
# We use outfmt 7 (tabular format with comment lines) to easily parse the outputs via scripts
blastp -query ../Domain_Validation/1.AmbTr_MIKCc_Final.fasta \
       -db AT_MADS_ref_db \
       -outfmt 7 \
       -evalue 1e-5 \
       -out AmbTr_vs_AT_blast.txt
```

> ### Step 2: Extract "Top Hit" for Annotation Logic
A raw BLAST output returns many similar hits. We need a script or awk command to solely extract the *#1 Ranked* ortholog per species-query ID to establish a 1-to-1 linkage for downstream functional prediction.
```bash
# This simple AWK script picks only the first instance of a query ID in standard outfmt 7
awk '!seen[$1]++' AmbTr_vs_AT_blast.txt > Amborella_trichopoda_TopHits_AT.txt
```

> ### Step 3: Execute Self-BLAST to Detect Redundant Isoforms
After successfully BLASTing against the AT database, we must perform a Self-BLAST recursively against the target species' own database to detect and remove identical splicing isoforms, redundancy, or sequences mapping to the exact same gene location.
```bash
# First construct a DB for the specific species
makeblastdb -in ../Domain_Validation/1.AmbTr_MIKCc_Final.fasta -dbtype prot -out AmbTr_SelfDB

# Then BLAST it against itself to detect 100% matched redundant isoforms
blastp -query ../Domain_Validation/1.AmbTr_MIKCc_Final.fasta \
       -db AmbTr_SelfDB \
       -outfmt 7 \
       -evalue 1e-5 \
       -out AmbTr_SelfBLAST.txt
```
*(Any 100% physically identical redundant isoform hits found here should be fully removed before building the phylogenetic tree).*
```bash
# Preview the output formatting
head -n 5 Amborella_trichopoda_TopHits_AT.txt
# Expected Format:
# Query_ID (Species)       Subject_ID (Arabidopsis)      %Identity   ... [E-Value]
# evm_27.model.AmTr...     AT5G20240.1                   45.6        ... 1e-50
```

*(This TSV Top-Hit matrix directly feeds into the later Subclade Python script!)*
