# <h2 style="color:darkblue; font:Times New Roman;">Phase 1: Data Acquisition — Step 01</h2> 
<br>

## **Aim**  
To construct the local directory environment for Phase 1 and organize the raw genome datasets retrieved from Phase 0 into a streamlined programmatic working directory. This prevents cluttering the initial database folder and safeguards the original downloads.
<br>

## Steps:-

```bash
#!/bin/bash
```

> ### Step 0: Construct Working Directories
We must first build the sandbox environment natively inside our NIPGR working directory.
```bash
cd /Users/bushrakhan/Desktop/NIPGR-data/NIPGR_WORK
mkdir -p Phase_1_Workspace
cd Phase_1_Workspace

mkdir -p Genomes Proteomes Annotations
```

> ### Step 1: Copy Original Genomes & Proteomes
Copy your Phase 0 downloaded datasets carefully into this working repository. This ensures if any script destructively parses sequences, your original datasets generated from `TASK_0` remain unharmed!
```bash
# Example syntax to securely copy the standardized species arrays
cp -r ../data/genomes/* Genomes/
cp -r ../data/proteomes/* Proteomes/
cp -r ../data/annotations/* Annotations/
```

> ### Step 2: Establish the Arabidopsis Reference System
Copy your TAIR10 files out of the baseline datasets directly here. We isolate *A. thaliana* separately from the structural genomic loops because it serves as the coordinate system rather than an identification target.
```bash
cp ../data/proteomes/Arabidopsis_TAIR10_proteins.fa .
mv Arabidopsis_TAIR10_proteins.fa Reference_Arabidopsis.fa
```

> ### Step 3: Validate Sequences
Verify that no sequence downloads were corrupted in transit.
```bash
# Count total sequences mapped
grep -c "^>" Proteomes/*.fa
```
