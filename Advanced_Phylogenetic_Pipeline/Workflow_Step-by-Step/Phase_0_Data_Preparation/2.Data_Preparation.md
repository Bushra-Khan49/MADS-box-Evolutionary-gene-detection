# <h2 style="color:darkblue; font:Times New Roman;">Phase 0: Data Preparation & Organization — Step 02</h2> 
<br>

## **Aim**  
To clean, standardize, and physically organize all the downloaded genome resources. It is essential to convert raw compressed `.gz` downloads into reliably indexed and structurally consistent datasets so that downstream BLAST, HMM, and Synteny workflows can securely parse them.
<br>

## Steps:-

```bash
#!/bin/bash
```

> ### Step 1: Set up the Directory Architecture
To keep exactly consistent with a professional pipeline, we move everything into mapped organizational folders physically decoupling raw inputs from parsed outputs.
```bash
mkdir -p data/genomes data/proteomes data/cds data/annotations data/domains
```

> ### Step 2: Unzip and Standardize Naming (Genomes & Proteins)
To prevent complex naming errors in downstream tools, we will unzip the raw files and forcefully standardize their base names across all lineages. 
```bash
# Decompress safely
gunzip *.fa.gz

# Rename structurally (e.g. Glycine max)
mv Gmax_assembly.fa glycine_max.genome.fa
mv Gmax_protein_primaryTranscriptOnly.fa glycine_max.protein.fa

# Move securely
mv *.genome.fa data/genomes/
mv *.protein.fa data/proteomes/
```

> ### Step 3: Indexing and Databases Preparation
Before a pipeline can query the genome computationally, we must index the genomic `.fa` blocks natively mapping them for the analysis frameworks.
```bash
# Construct standard genomic indices
cd data/genomes
samtools faidx glycine_max.genome.fa

# Construct the all-important Proteome BLAST DB which is used constantly
cd ../proteomes
makeblastdb -in glycine_max.protein.fa -dbtype prot
```

> ### Step 4: Structuring the Essential Annotations (GFF3)
Annotations natively contain the distinct coordinates mapping a gene explicitly to the genome. Many downstream tools severely crash unless the GFF3 coordinate maps are sorted ascending!
```bash
cd ../annotations
gunzip *.gff3.gz

# Sort structurally by chromosome and positional mapping
sort -k1,1 -k4,4n glycine_max.gff3 > sorted.gff3
mv sorted.gff3 glycine_max.gff3
```

> ### Step 5: Preparing the Domain Ensembles
Move the three downloaded HMM profiles (MADS, K-box, and AP2) into your `domains` folder, and logically press them so `hmmsearch` can natively invoke them later.
```bash
cd ../domains
mv *.hmm .

# Press the raw mathematical seeds
hmmpress PF00319.hmm
hmmpress PF01486.hmm
hmmpress PF00847.hmm
```
