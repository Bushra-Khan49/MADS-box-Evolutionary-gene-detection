
# Step 2. Data Preparation & Organization

---

## Aim

To clean, standardize, and organize all downloaded genome resources so they are ready for downstream comparative genomics and evolutionary analyses. This step converts raw downloads into indexed, searchable, and pipeline-compatible datasets.


---

# Concept

### Step 1 → Download
### Step 2 → Clean + Organize + Standardize + Prepare for analysis

Each file type requires specific preparation before it can be used reliably in BLAST, HMM searches, phylogenetics, or synteny workflows.

---


## Required Files (Core Processing)




## 1. Genome file (.fa / .fna)

#### Examples

```bash 
Gmax_assembly.fa.gz
Athaliana.fa.gz
Amborella.fa.gz
```

### Steps

#### Step 2A — Unzip

```bash 
gunzip *.fa.gz
```

#### Step 2B — Rename (important for scripts)

```bash 
mv Gmax_assembly.fa glycine_max.genome.fa
```

#### Step 2C — Index

```bash 
samtools faidx glycine_max.genome.fa
```

Optional (for BLAST)

```bash 
makeblastdb -in glycine_max.genome.fa -dbtype nucl
```

Used later for
- Synteny analysis
- Gene coordinate extraction
- Genome visualization
- Sequence mapping




## 2. Protein file (protein_primaryTranscriptOnly.fa)

Most important file in the entire pipeline

#### Example

```bash 
Gmax_protein_primaryTranscriptOnly.fa
```

#### Step 2A — Unzip

```bash 
gunzip *.fa.gz
```

#### Step 2B — Rename

```bash 
mv Gmax_protein_primaryTranscriptOnly.fa glycine_max.protein.fa
```

#### Step 2C — Clean headers (recommended)

```bash 
seqkit replace -p " .*" -r "" glycine_max.protein.fa > clean.fa
mv clean.fa glycine_max.protein.fa
```

#### Step 2D — Make BLAST database

```bash 
makeblastdb -in glycine_max.protein.fa -dbtype prot
```

Used later for
- HMM domain search
- BLASTp
- OrthoFinder
- Phylogenetic tree construction
- Domain analysis
  



## 3. Annotation file (.gff3)

#### Example

```bash 
glycine_max.gff3.gz
```

#### Step 2A — Unzip

```bash 
gunzip *.gff3.gz
```

#### Step 2B — Sort (required for many tools)

```bash 
sort -k1,1 -k4,4n glycine_max.gff3 > sorted.gff3
mv sorted.gff3 glycine_max.gff3
```

#### Step 2C — Optional BED conversion

```bash 
gff2bed < glycine_max.gff3 > glycine_max.bed
```

Used later for
- Gene coordinates
- Synteny
- Sequence extraction
- Genome mapping




## 4. Pfam HMM domain files (.hmm)

#### Examples

```bash 
PF00319.hmm
PF01486.hmm
PF00847.hmm
```

#### Step 2A — Move to domains folder

```bash 
mkdir -p data/domains
mv *.hmm data/domains/
```

#### Step 2B — Prepare database

```bash 
hmmpress PF00319.hmm
hmmpress PF01486.hmm
hmmpress PF00847.hmm
```

Used later for

```bash 
hmmsearch
```

## Optional Files (process only if needed)


## 5. CDS file (.cds.fa) — optional

### Steps

```bash 
gunzip *.gz
mv file.cds.fa glycine_max.cds.fa
samtools faidx glycine_max.cds.fa
```

Used for
- Back-translation
- Codon analysis
- Ka/Ks calculations
- Validation


## 6. Transcript file (.cdna.fa / .mrna.fa) — optional

### Steps

```bash 
gunzip *.gz
```

(No further processing required)

Used for

- RNA-seq mapping
- Expression analysis

## 🗂️ Final Organization

#### Recommended professional directory structure

```bash 
data/
 ├── genomes/
 ├── proteomes/
 ├── cds/
 ├── transcripts/
 ├── annotations/
 ├── domains/
 ├── blastdb/
```

### 📥 Move files

```bash 
mv *.genome.fa genomes/
mv *.protein.fa proteomes/
mv *.gff3 annotations/
mv *.hmm domains/
```

## 🔖 Quick Cheat Sheet

- File Type	Action - Why?
- Genome	unzip + index	synteny
- Protein	unzip + BLAST db	main analysis
- GFF3	sort	coordinates
- HMM	hmmpress	domain search
- CDS	unzip + index	optional validation
- Transcript	store only	RNA-seq
