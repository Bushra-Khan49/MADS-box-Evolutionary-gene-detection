## Step-by-Step Workflow
Run commands sequentially in your terminal.
 
Each step includes the command, input/output files, and notes. 

Inspection commands (e.g., `ls -l`, `head`) are optional for verification.

### Step 1: Inspect Initial Directory
```bash
pwd
ls -l
```
- **Inputs**: None.
- **Outputs**: Terminal display of files.
- **Notes**: Confirms presence of input FASTA files.

### Step 2: Build Prunus Protein BLAST Database
```
makeblastdb -in Ppersica_298_v2.1.protein.fa -dbtype prot -out Ppersica_protDB
```
- **Inputs**:Ppersica_298_v2.1.protein.fa.
- **Outputs**:  Database files (Ppersica_protDB.* – e.g., .phr, .pin, .psq).
- **Notes**:  Adds 47,089 sequences. Takes ~1 second.

### Step 3 : Inspect Post-DB Directory
```
ls -l
```
- **Notes**: Verify DB files created.

### Step 4 : Forward BLAST ( Tabular Format 7 -with comments) 
```
blastp -query AT_ALL_MADSbox_PROTEINseq.fa -db Ppersica_protDB \
  -out AT_MADS_vs_Ppersica.blastp.out -evalue 1e-5 -outfmt 7 -num_threads 8
```
- **Inputs**: AT_ALL_MADSbox_PROTEINseq.fa, Ppersica_protDB.
- **Outputs**: AT_MADS_vs_Ppersica.blastp.out (327,593 bytes).
- **Notes**: E-value threshold: 1e-5. Uses 8 threads.

### Step 5: Inspect Post-BLAST
```
ls -l
```
- **Notes**: Verify BLAST output.

### Step 6: Forward BLAST (Tabular Format 6 – clean tabular)
```
blastp -query AT_ALL_MADSbox_PROTEINseq.fa -db Ppersica_protDB \
  -out AT_MADS_vs_Ppersica2.blastp.out -evalue 1e-5 -outfmt 6 -num_threads 8
```
- **Inputs**: Same as Step 4.
- **Outputs**: AT_MADS_vs_Ppersica2.blastp.out (293,104 bytes).
- **Notes**: Cleaner format for downstream processing.

### Step 7: Inspect Post-Second BLAST
```
ls -l
```
- **Notes**: Verify output.

### Step 8: Remove Duplicate BLAST Lines
```
sort AT_MADS_vs_Ppersica2.blastp.out | uniq > AT_MADS_vs_Ppersica.blastp.nodup.out
```
- **Inputs**: AT_MADS_vs_Ppersica2.blastp.out.
- **Outputs**: AT_MADS_vs_Ppersica.blastp.nodup.out (293,104 bytes).
- **Notes**: Sorts and deduplicates exact lines.

### Step 9: Extract Query-Subject Pair
```
awk '{print $1"\t"$2}' AT_MADS_vs_Ppersica.blastp.nodup.out > query_subject_pairs.txt
```
- **Inputs**: AT_MADS_vs_Ppersica.blastp.nodup.out.
- **Outputs**: query_subject_pairs.txt (120,147 bytes).
- **Notes**: Tab-separated: AT_ID (col1) and Prupe_ID (col2).

### Step 10: Extract Unique Prunus Subject IDs
```
cut -f2 query_subject_pairs.txt | sort -u > subject_ids.txt
```
- **Inputs**: query_subject_pairs.txt.
- **Outputs**: subject_ids.txt (1,258 bytes; ~1,258 unique IDs).
- **Notes**: List of candidate Prunus hits.

### Step 11: Inspect Pairs File
```
ls -l
head query_subject_pairs.txt
```
- **Notes**: Quick check (e.g., shows pairs like AT1G01530.1	Prupe.1G032200.1).

### Step 12: Extract Prunus Hit Sequences
```
seqtk subseq Ppersica_298_v2.1.protein.fa subject_ids.txt > subject_seqs.fa
```
- **Inputs**: Ppersica_298_v2.1.protein.fa, subject_ids.txt.
- **Outputs**: subject_seqs.fa (32,798 bytes).
- **Notes**: Subsets FASTA for hits.

### Step 13: Inspect Extracted Sequences
```
head subject_seqs.fa
ls -l
```
- **Notes**: Preview sequences (some may have stop codons *).

### Step 14: Label Sequences with AT|Prupe Headers
```
awk 'NR==FNR{map[$2]=$1"|" $2; next} /^>/{id=substr($1,2); if(id in map){print ">"map[id]} else {print $0}} !/^>/' \
  query_subject_pairs.txt subject_seqs.fa > hits_sequences_labeled.fa
```
- **Inputs**: query_subject_pairs.txt, subject_seqs.fa.
- **Outputs**: hits_sequences_labeled.fa (27,766 bytes).
- **Notes**: Relabels headers (e.g., >AT5G65080.2|Prupe.2G109500.1).

### Step 15: Inspect Labeled Sequences
```
head hits_sequences_labeled.fa
ls -l
```
- **Notes**: Verify labeling.

### Step 16: Reverse BLAST (RBH Check)
```
blastp -query hits_sequences_labeled.fa -db Ppersica_protDB \
  -out hits_vs_Ppersica.blastp.out -evalue 1e-5 -outfmt 6 -num_threads 8
```
- **Inputs**: hits_sequences_labeled.fa, Ppersica_protDB.
- **Outputs**: hits_vs_Ppersica.blastp.out (297,157 bytes; 3,360 lines).
- **Notes**: Searches labeled hits back; expects self-hits for reciprocity.

### Step 17: Inspect Reverse BLAST Results
```
head hits_vs_Ppersica.blastp.out
ls -l
wc -l hits_vs_Ppersica.blastp.out
cut -f1 hits_vs_Ppersica.blastp.out | sort -u | wc -l  # 74 unique queries
cut -f2 hits_vs_Ppersica.blastp.out | sort -u | wc -l  # 94 unique subjects
head -n 20 hits_vs_Ppersica.blastp.out
```
- **Notes**: Many 100% identity self-hits.

### Step 18: Select Best Hit per Query (RBH)
```
awk 'BEGIN{FS=OFS="\t"} {
  q=$1; e=$11+0; bs=$12+0;
  if(!(q in ebest) || e<ebest[q] || (e==ebest[q] && bs>bbest[q])) {
    ebest[q]=e; bbest[q]=bs; best[q]=$0
  }
}
END{for(q in best) print best[q]}' hits_vs_Ppersica.blastp.out > hits_vs_Ppersica.besthit.out
```
- **Inputs**: hits_vs_Ppersica.blastp.out.
- **Outputs**: hits_vs_Ppersica.besthit.out (6,220 bytes; 74 lines).
- **Notes**: Prioritizes lowest e-value, then highest bitscore.

### Step 19: Inspect Best Hits
```
wc -l hits_vs_Ppersica.besthit.out
head hits_vs_Ppersica.besthit.out
ls -l
```
- **Notes**: Confirms 74 RBH (mostly self-hits).

### Step 20: Extract Best Hit Prunus IDs
```
cut -f2 hits_vs_Ppersica.besthit.out | sort -u > best_hit_subject_ids.txt
```
- **Inputs**: hits_vs_Ppersica.besthit.out.
- **Outputs**: best_hit_subject_ids.txt (1,258 bytes; 74 IDs).
- **Notes**: Unique Prunus ortholog IDs.

### Step 21: Inspect Best Hit IDs
```
wc -l best_hit_subject_ids.txt
head best_hit_subject_ids.txt
```
- **Notes**: List like Prupe.1G032200.1.

### Step 22: Extract Best Hit Sequences (Original Headers)
```
seqtk subseq Ppersica_298_v2.1.protein.fa best_hit_subject_ids.txt > best_hit_subject_seqs.fa
```
- **Inputs**: Ppersica_298_v2.1.protein.fa, best_hit_subject_ids.txt.
- **Outputs**: best_hit_subject_seqs.fa (32,798 bytes).
- **Notes**: Full Prunus headers.

### Step 23: Inspect Best Hit Sequences
```
head best_hit_subject_seqs.fa
ls -l
```
- **Notes**: Preview.

### Step 24: Create Prunus-Only FASTA (Stripped Labels)
```
awk '/^>/{split($0,a,"|"); print ">"a[2]; next} {print}' hits_sequences_labeled.fa > hits_sequences_prunus_only.fa
```
- **Inputs**: hits_sequences_labeled.fa.
- **Outputs**: hits_sequences_prunus_only.fa (26,878 bytes).
- **Notes**: Headers like >Prupe.2G109500.1.

### Step 25: Subset to Best Hits (Prunus-Only)
```
seqtk subseq hits_sequences_prunus_only.fa best_hit_subject_ids.txt > best_hit_seqs_labeled.fa
```
- **Inputs**: hits_sequences_prunus_only.fa, best_hit_subject_ids.txt.
- **Outputs**: best_hit_seqs_labeled.fa (26,878 bytes).
- **Notes**: RBH sequences with Prunus headers.

### Step 26: Inspect Subset
```
head best_hit_seqs_labeled.fa
```
- **Notes**: Quick check.

### Step 27: Generate Prunus-to-AT Mapping
```
awk -F'[>|]' '/^>/{print $3"\t"$2}' hits_sequences_labeled.fa > prunus2at_mapping.txt
```
- **Inputs**: hits_sequences_labeled.fa.
- **Outputs**: prunus2at_mapping.txt (2,146 bytes).
- **Notes**: Tab-separated: Prupe_ID	AT_ID.

### Step 28: Inspect Mapping
```
head prunus2at_mapping.txt
```
- **Notes**: E.g., Prupe.2G109500.1	AT5G65080.2.

### Step 29: Relabel RBH Sequences with AT|Prupe
```
awk 'BEGIN{while(getline<"prunus2at_mapping.txt") map[$1]=$2}
/^>/{split($0,a," "); id=a[1]; sub(/^>/,"",id); if(id in map) print ">"map[id]"|"id; else print $0; next}
{print}' best_hit_seqs_labeled.fa > best_hit_seqs_AT_labeled.fa
```
- **Inputs**: prunus2at_mapping.txt, best_hit_seqs_labeled.fa.
- **Outputs**: best_hit_seqs_AT_labeled.fa (27,766 bytes).
- **Notes**: Final ortholog FASTA with dual labels.

### Step 30: Inspect Final Output
```
head best_hit_seqs_AT_labeled.fa
ls -l
```
