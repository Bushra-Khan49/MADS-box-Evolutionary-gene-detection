# <h2 style="color:darkblue; font:Times New Roman;">Phase 1: HMMER Profiling — Step 02</h2> 
<br>

## **Aim**  
Scan the bulk proteome data to conditionally extract only sequences containing the highly conserved MADS-box DNA binding domain utilizing Hidden Markov Models (HMMER) and the established SRF-TF (PF00319) Pfam profile. This ensures rigorous screening against our targeted structural threshold.
<br>

## Steps:-

```bash
#!/bin/bash
```

> ### Step 1: Execute `hmmsearch` against All Proteomes
Use the mathematical HMM profile to structurally search the massive target species proteome files mathematically. We set a strict statistical E-value threshold (`1e-5`) to force the model to aggressively avoid false positive homologs.

Here is the exact executable logic mapped across varying lineages:

```bash
cd /Users/bushrakhan/Desktop/NIPGR-data/NIPGR_WORK/Phase_1_Workspace
HMM_FILE="../data/domains/PF00319.hmm"

# Amborella trichopoda (Basal Angiosperm)
hmmsearch --cut_ga --domE 1e-5 -E 1e-5 -o AmbTr_HMM_output.txt --tblout AmbTr_hits.tbl $HMM_FILE Proteomes/AmbTr.fa

# Cinnamomum kanehirae (Magnoliid)
hmmsearch --cut_ga --domE 1e-5 -E 1e-5 -o CinKa_HMM_output.txt --tblout CinKa_hits.tbl $HMM_FILE Proteomes/CinKa.fa

# Oryza sativa (Monocot)
hmmsearch --cut_ga --domE 1e-5 -E 1e-5 -o OrySat_HMM_output.txt --tblout OrySat_hits.tbl $HMM_FILE Proteomes/OrySat.fa

# Medicago truncatula (Rosid)
hmmsearch --cut_ga --domE 1e-5 -E 1e-5 -o Mtrunc_HMM_output.txt --tblout Mtrunc_hits.tbl $HMM_FILE Proteomes/Mtrunc.fa

# Glycine max 
hmmsearch --cut_ga --domE 1e-5 -E 1e-5 -o GlyMax_HMM_output.txt --tblout GlyMax_hits.tbl $HMM_FILE Proteomes/Glycine_max.protein.fa

# Follow suit iteratively for Prunus persica, Nymphaea colorata, Nelumbo nucifera, and Helianthus annuus!
```

> ### Step 2: Extract Hit Sequence IDs & Remove Duplicates
Extract just the raw Gene IDs from the dense HMMER tabular output to cleanly prepare for sequence pulling. It is absolutely critical to explicitly enforce string duplicate removal here so that redundant multi-domain structural hits on the exact same protein are collapsed.

```bash
# The 'sort | uniq' pipeline ensures any multi-domain duplicate hits on the same gene are cleanly merged.
grep -v "^#" AmbTr_hits.tbl | awk '{print $1}' | sort | uniq > AmbTr_MADS_IDs.txt
grep -v "^#" CinKa_hits.tbl | awk '{print $1}' | sort | uniq > CinKa_MADS_IDs.txt
grep -v "^#" OrySat_hits.tbl | awk '{print $1}' | sort | uniq > OrySat_MADS_IDs.txt

# Verify the collapsed candidate sizes
wc -l *_MADS_IDs.txt
```

> ### Step 3: Pull FASTA Sequences for Putative Hits
Use `seqtk` to physically rip the targeted protein sequence strings arrays from the raw original FASTA files solely matching our generated filtered IDs list. 

```bash
# seqtk subseq [in.fasta] [id_list.txt] > [out.fasta]
seqtk subseq Proteomes/AmbTr.fa AmbTr_MADS_IDs.txt > AmbTr_putative_MADS.fa
seqtk subseq Proteomes/CinKa.fa CinKa_MADS_IDs.txt > CinKa_putative_MADS.fa
seqtk subseq Proteomes/OrySat.fa OrySat_MADS_IDs.txt > OrySat_putative_MADS.fa

# Validate generated sets visually
ls -l *_putative_MADS.fa
```

*(You now possess localized, deduplicated FASTA sets strictly containing MADS-domain bearing domains!)*
