# <h2 style="color:darkblue; font:Times New Roman;">Phase 1: Structural Domain Validation (Type II Isolation) — Step 03</h2> 
<br>

## **Aim**  
Filter out simpler Type I (SRF-like) MADS-box genes by performing a rigorous structural check using InterProScan/Pfam to confirm the mandatory co-presence of the Type II identifying "K-box" protein interaction domain (PF01486).
<br>

## Steps:-

```bash
#!/bin/bash
```

> ### Step 0: Upload putative FASTA hits to Domain Scanner
While large clusters can be run locally using `InterProScan`, smaller or batched datasets can be processed using the EMBL-EBI InterProScan 5 endpoint or Pfam batch sequence search.
```bash
# Example local execution of standalone InterProScan
cd /Users/bushrakhan/Desktop/NIPGR-data/NIPGR_WORK
mkdir Domain_Validation
cd Domain_Validation

./interproscan.sh -i ../HMM_Profiling/AmbTr_putative_MADS.fa -f tsv -appl Pfam
```

> ### Step 1: Parse the PFAM results for the K-box signature
The output `.tsv` files natively contain Pfam domain matches for every single submitted gene.
```bash
# Look for sequences that possess both PF00319 (MADS) and PF01486 (K-box)
grep "PF01486" AmbTr_putative_MADS.fa.tsv | awk '{print $1}' | sort | uniq > AmbTr_Kbox_IDs.txt
```

> ### Step 2: AP2 Domain Filtering
To ensure no overlapping or pseudo-MADS gene sequences are retained natively tied to AP2 complex networks, we explicitly grep and purge any ID containing the AP2 domain (PF00847).
```bash
# Identify any IDs possessing PF00847
grep "PF00847" AmbTr_putative_MADS.fa.tsv | awk '{print $1}' | sort | uniq > AmbTr_AP2_IDs.txt

# Remove those AP2-containing IDs from our true list using grep reversed matching
grep -v -f AmbTr_AP2_IDs.txt AmbTr_Kbox_IDs.txt > AmbTr_True_MIKCc_IDs.txt
```

> ### Step 3: Extract Finalized True MIKCc Protein Subsets
Now we know exactly which IDs are structurally valid Type II (MIKCc) transcription factors (without any AP2 contamination). We explicitly reconfirm these proteins possess the correct functional domains. If any proteins lack these domains, or conversely have too many undesired functional domains (and are uncharacteristically large), they are safely removed. 

We will pull their full sequences into a final, pristine working FASTA file for our subsequent evolutionary study (keeping only one sequence per redundant protein block).
```bash
seqtk subseq ../Raw_Proteomes/AmbTr.fa AmbTr_True_MIKCc_IDs.txt > 1.AmbTr_MIKCc_Final.fasta
```
